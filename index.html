Understood. Staying abstract. The IR/runtime concept is: AI output is not “the thing”; it is a representation interpreted by some runtime. Other abstractions at the same level:

1. AI as compression

A model can be viewed as a huge compression artifact. It compresses patterns from training data into weights, then reconstructs plausible continuations from a prompt.

It does not store the world literally — it stores a compressed statistical structure of the world. This explains both power and failure: it can generalize because compression captures patterns; it hallucinates because reconstruction is not retrieval.

intelligence as lossy compression plus reconstruction

2. AI as conditional simulation

A prompt does not command the model in a mechanical sense. It conditions a simulator — the model simulates the kind of text, reasoning, persona, task, or world implied by the input.

When the prompt says “you are a careful mathematician,” the model enters that distribution. When it says “write a pirate song,” it enters another. This is why framing is so powerful: you are not merely giving instructions, you are setting initial conditions for a simulation.

prompting as world-state initialization

3. AI as distribution shaping

The model always has many possible next outputs. Sampling, temperature, system prompts, examples, schemas, and constraints reshape the probability distribution. This abstraction is more fundamental than “asking questions” — you are sculpting the probability landscape.

  • A schema narrows the distribution.
  • A style example bends it.
  • A low temperature sharpens it; a high temperature widens it.
  • A tool list changes what outputs are likely.
  • A prior answer becomes momentum.

AI behavior as probability mass being pushed around

4. AI as latent-space navigation

The model maps inputs into internal representations. Similar ideas cluster near each other. Reasoning can be seen as movement through this latent space: from vague intent, to concepts, to candidate structures, to output.

This is why analogies work. The model can move from “Markdown is a display language” to “JSON tool calls are a display/action language” because those structures are nearby in conceptual space.

thinking as movement through compressed concept geometry

5. AI as attention allocation

Attention is not just a transformer mechanism — it is a system-level concept. Any intelligent process must decide what matters now.

In AI, attention exists at multiple levels:

  • The model attends to tokens.
  • The prompt attends to selected facts.
  • The system attends to certain tools.
  • The user attends to certain outputs.
  • The evaluator attends to certain success criteria.

A lot of intelligence is not “knowing more.” It is attending to the right subset.

intelligence as selective relevance

6. AI as search

Generation can be understood as search through possible continuations, plans, answers, programs, and explanations. The model proposes candidates; decoding chooses among them; external systems can expand, rank, prune, retry, or verify.

This abstraction explains why “one-shot answer” is weak. A stronger system treats output as candidate generation inside a search process.

AI as proposal engine inside a search space

7. AI as amortized reasoning

A model often appears to reason instantly because it has compressed many previous reasoning patterns into its weights. Instead of solving every problem from scratch, it uses learned shortcuts.

That is amortization: expensive training produces cheap inference. This is why models can be fast and impressive, but brittle on problems outside their learned distribution.

training stores reusable cognitive shortcuts

8. AI as translation between representations

AI is often not “answering” — it is translating between representations:

  • Natural language to code
  • Image to caption
  • Question to SQL
  • Messy notes to outline
  • Goal to plan, plan to action
  • Error log to diagnosis
  • Data to explanation
  • Text to UI, UI to state

This is broader than the IR/runtime idea. The key is not execution — the key is representation change.

intelligence as representation transformation

9. AI as constraint satisfaction

A model is constantly satisfying constraints:

  • The grammar must work.
  • The answer must fit the prompt.
  • The style must match; the facts should cohere.
  • The schema must validate; the task goal should be met.
  • The next token must be likely.

More constraints usually produce more useful output, as long as they are not contradictory. This reframes prompting as constraint design.

prompting as constraint programming in natural language

10. AI as an uncertainty engine

A model does not truly “know” one answer internally — it has distributions over possible answers. Confidence, ambiguity, hallucination, creativity, and disagreement all come from this.

Bad systems collapse uncertainty too early. Good systems preserve uncertainty until verification or choice. Examples:

  • “Maybe these three causes.”
  • “Here are candidate interpretations.”
  • “This field is unknown.”
  • “This action needs confirmation.”
  • “These outputs have ranked likelihood.”

intelligence as managing uncertainty, not eliminating it

11. AI as a critic

A model does not only generate — it can evaluate. It can score whether an answer follows instructions, whether a plan is plausible, whether a UI component matches intent, whether a command looks risky, whether a summary preserves meaning.

Generation and criticism are different roles. The same architecture can be used as a proposer or as a judge, but those are different abstractions.

AI as generator versus AI as discriminator

12. AI as feedback control

Once AI interacts with the world, the key abstraction becomes control. There is a goal, an action, an observation, and an error signal. The system acts again to reduce the gap between current state and desired state.

This is different from text generation — it is closed-loop behavior. A non-agentic model produces an output; a control system keeps adjusting until the state changes.

agency as closed-loop error correction

13. AI as policy

A policy maps states to actions. This abstraction comes from reinforcement learning, but it applies broadly: given the current situation, what should be done next?

  • A language model can be seen as a policy over tokens.
  • An agent model can be seen as a policy over actions.
  • A router can be seen as a policy over modules.
  • A UI model can be seen as a policy over components.

intelligence as choosing the next move

14. AI as affordance detector

An affordance is a possible action implied by the environment:

  • A button affords clicking.
  • A text box affords typing.
  • A file affords opening; a sentence affords summarizing.
  • A table affords querying; a contradiction affords correction.

Models are useful when they detect what can be done with a given object or situation.

understanding as seeing possible operations

15. AI as ontology builder

The model does not just process facts — it carves the world into categories. It decides what the objects are, what relations matter, what level of description is appropriate, and what distinctions are meaningful.

Before solving anything, the system frames the domain. Is this a planning problem? A debugging problem? A design problem? A routing problem? A classification problem? A conflict between constraints?

intelligence as choosing the right ontology

16. AI as compression of procedures into skills

Repeated behavior can be compressed into a reusable unit. At first, the system reasons step by step; later, the same pattern becomes a skill, macro, function, habit, or policy.

This is how intelligence becomes efficient — deliberate reasoning hardens into automatic procedure.

learning as turning deliberation into reusable action

17. AI as boundary negotiation

AI systems constantly negotiate boundaries:

  • What is inside the model versus outside?
  • What is language versus action?
  • What is user intent versus system policy?
  • What is memory versus context?
  • What is suggestion versus execution?
  • What is uncertain versus confirmed?
  • What is generated versus retrieved?

Many failures come from bad boundary placement.

system design as deciding where boundaries live

18. AI as scale-shifting

AI can move between levels of abstraction — summarize a thousand details into one concept, expand one concept into detailed steps, turn a principle into examples, examples into a rule, a rule into a procedure.

This is one of the most important general capabilities.

intelligence as moving between compression levels

19. AI as style-transfer over meaning

Style is not superficial — it changes which features are emphasized:

  • A legal style foregrounds definitions and risk.
  • A technical style foregrounds mechanism.
  • A poetic style foregrounds association.
  • A terse style foregrounds selection.
  • A tutorial style foregrounds sequence.

The same content passed through different styles becomes different cognitive machinery.

style as an information filter

20. AI as meaning-preserving distortion

Every model output distorts the input. The question is whether the distortion preserves what matters:

  • Summarization distorts by compression.
  • Translation distorts across languages.
  • Explanation distorts by simplification.
  • Classification distorts by discretization.
  • Visualization distorts by projection.
  • Analogy distorts by mapping.

AI transforms information by controlled distortion

21. AI as synthetic sensor

A model can act like a sensor for things that are not directly measured — tone, intent, topic, risk, ambiguity, contradiction, relevance, salience, sentiment, genre, structure, or implied goal.

These are not physical signals. They are semantic signals inferred from data.

AI as a sensor for latent properties

22. AI as semantic glue

AI is useful because it can connect systems that were not designed to talk to each other:

  • Humans speak loosely; machines require formality.
  • Databases have schemas; UIs have components.
  • Files have formats; APIs have contracts.

AI sits between incompatible representations and makes them interoperable.

AI as adapter between mismatched worlds

23. AI as entropy manager

Raw possibility is high entropy. A model reduces it — a blank page has infinite possibilities; a prompt narrows them; a schema narrows them further; a plan narrows them into sequence; a final answer collapses them into one path.

Creativity increases useful entropy; execution reduces entropy. AI systems alternate between expanding and collapsing possibility.

AI systems alternate between expanding and collapsing possibility

24. AI as a mirror with transformation

A model often reflects the structure of the input back at you, but transformed:

  • Ask vaguely, get vague shape back.
  • Ask with structure, get structure back.
  • Ask with assumptions, get those assumptions amplified unless challenged.
  • Provide examples, get pattern continuation.

This is not just “garbage in, garbage out.” It is more like the model amplifies the latent structure of the input.

the model amplifies the latent structure of the input

25. AI as a coordination layer

AI can coordinate between agents, tools, documents, users, tasks, goals, and representations. It does not necessarily do the work itself — it decides who or what should do which part, in what order, under which constraints.

This is orchestration at the semantic level.

AI as task decomposition and delegation machinery

AI abstraction patterns 02 — major examples

Source: localsend/ai-abstractions-patterns-02-list-major-examples.txt

1. Compression and reconstruction

AI compresses patterns into a smaller internal form, then reconstructs useful outputs from that compressed structure.

Example

Summarizing a 50-page report into a one-page brief, then expanding the brief back into a slide deck.

2. Conditional simulation

A prompt sets the conditions for what kind of world, role, style, or task the model simulates.

Example

"Act like a senior Linux engineer debugging an i3 window manager issue" produces a different reasoning mode than "explain this to a beginner."

3. Distribution shaping

Prompts, examples, schemas, temperature, and tool choices reshape what outputs are likely.

Example

Giving a JSON schema makes the model much more likely to produce valid machine-readable actions instead of loose prose.

4. Latent-space navigation

AI moves through compressed concept-space, connecting nearby meanings, analogies, formats, and structures.

Example

Seeing that "a browser tab," "a document," "a window," and "an app state" can all be treated as navigable objects.

5. Attention allocation

Intelligence often means deciding what matters right now and ignoring the rest.

Example

In a long error log, the model identifies the three lines that actually explain the crash.

6. Search over possibilities

AI generates candidates, compares them, prunes weak options, and refines stronger ones.

Example

Producing five possible architectures for a local agent harness, then selecting the simplest one that satisfies the constraints.

7. Amortized reasoning and skill compression

Training turns expensive repeated reasoning into fast reusable patterns. Repeated workflows can also become tools, macros, or skills.

Example

Instead of reasoning from scratch every time, a coding assistant learns the common repair pattern for a broken import, missing dependency, or bad config path.

8. Representation translation

AI converts one representation into another: text to code, image to caption, prompt to UI, notes to outline, error log to diagnosis.

Example

A messy spoken command becomes structured JSON, which becomes an i3 window action.

9. Media transformation

Digital media is increasingly one continuous substrate. An image is simultaneously a book, movie, audio track, website, app, interface, dataset, or product mockup, because each form is now only a prompt or transformation away.

Example

A single concept image can become a storyboard, a narrated video, a landing page, an interactive prototype, a children's book, or a 3D asset prompt.

10. Constraint satisfaction

Prompting is often constraint design: format, tone, facts, schema, length, safety, tool limits, and goal all compete.

Example

"Write this as valid YAML, under 40 lines, with no optional fields, compatible with systemd" forces a much better output than "make a config."

11. Uncertainty management

Good AI systems preserve ambiguity until they can verify or decide. Bad systems collapse uncertainty too early.

Example

A diagnostic assistant says: "Most likely A, possible B, unlikely C; run this command to distinguish them."

12. Generator / critic separation

One system proposes; another evaluates. These are different jobs even when done by the same model.

Example

One model writes code, another checks whether it compiles, follows the spec, and avoids destructive commands.

13. Feedback control

An agent becomes useful when it observes the result of an action and corrects course.

Example

The system opens a file, sees it was the wrong one, searches again, narrows the query, and opens the correct file.

14. Policy over actions

A policy maps the current state to the next move.

Example

Given "YouTube is already open, video is playing, user says 'full screen,'" the system chooses the fullscreen action instead of launching a new browser.

15. Affordance detection

AI detects what can be done with something.

Example

It sees a spreadsheet and infers: sort, filter, summarize, validate formulas, detect outliers, generate charts.

16. Ontology formation

AI frames the problem by deciding what the objects, relationships, and categories are.

Example

A vague request like "my automation stack is getting messy" becomes objects such as router, executor, skill registry, fallback model, logs, tests, and UI surface.

17. Boundary placement

System design depends on deciding what belongs inside the model, outside the model, in memory, in tools, in code, or in the runtime.

Example

Let the model choose intent, but let deterministic code execute the window move.

18. Scale-shifting

AI moves up and down levels of abstraction.

Example

It can turn 200 implementation details into one principle, then turn that principle back into concrete commands, files, or UI components.

19. Style as filtering

Style changes what information is emphasized.

Example

The same incident report written in legal style emphasizes liability; in engineering style, root cause; in executive style, impact and next action.

20. Controlled distortion

Every transformation changes the input. The question is whether it preserves what matters.

Example

A diagram simplifies reality, but if it preserves the causal structure, the distortion is useful.

21. Synthetic sensing

AI can detect latent properties that are not directly measured: tone, intent, salience, risk, ambiguity, contradiction, relevance.

Example

A support bot detects that a user's message is not just a question but an escalation risk.

22. Semantic glue

AI connects systems that were not designed to talk to each other.

Example

A human says, "Put my research notes into the client format," and the model bridges loose prose, file structure, template rules, and final document format.

23. Entropy management

AI alternates between expanding possibility and collapsing it into action.

Example

Brainstorm 20 product names, narrow to five, test against constraints, choose one.

24. Structure amplification

Models amplify the structure already present in the input.

Example

Give rough bullets and it produces a cleaner outline; give messy assumptions and it may amplify the mess unless challenged.

25. Coordination and delegation

AI can decide which tool, model, person, document, or process should handle each part of a task.

Example

A research agent uses search for facts, a parser for PDFs, a model for synthesis, a critic for verification, and a formatter for final output.
