Staying abstract. The IR/runtime concept is: AI output is not “the thing”; it is a representation interpreted by some runtime. Other abstractions at the same level: 1. AI as compression A model can be viewed as a huge compression artifact. It compresses patterns from training data into weights, then reconstructs plausible continuations from a prompt. It does not store the world literally. It stores a compressed statistical structure of the world. This explains both power and failure. It can generalize because compression captures patterns. It hallucinates because reconstruction is not retrieval. The abstract form: intelligence as lossy compression plus reconstruction 2. AI as conditional simulation A prompt does not command the model in a mechanical sense. It conditions a simulator. The model simulates the kind of text, reasoning, persona, task, or world implied by the input. When the prompt says “you are a careful mathematician,” the model enters that distribution. When the prompt says “write a pirate song,” it enters another. This is why framing is so powerful. You are not merely giving instructions. You are setting initial conditions for a simulation. prompting as world-state initialization 3. AI as distribution shaping The model always has many possible next outputs. Sampling, temperature, system prompts, examples, schemas, and constraints reshape the probability distribution. This abstraction is more fundamental than “asking questions.” You are sculpting the probability landscape. A schema narrows the distribution. A style example bends it. A low temperature sharpens it. A high temperature widens it. A tool list changes what outputs are likely. A prior answer becomes momentum. AI behavior as probability mass being pushed around 4. AI as latent-space navigation The model maps inputs into internal representations. Similar ideas cluster near each other. Reasoning can be seen as movement through this latent space: from vague intent, to concepts, to candidate structures, to output. This is why analogies work. The model can move from “Markdown is a display language” to “JSON tool calls are a display/action language” because those structures are nearby in conceptual space. thinking as movement through compressed concept geometry 5. AI as attention allocation Attention is not just a transformer mechanism. It is a system-level concept. Any intelligent process must decide what matters now. In AI, attention exists at multiple levels: The model attends to tokens. The prompt attends to selected facts. The system attends to certain tools. The user attends to certain outputs. The evaluator attends to certain success criteria. A lot of intelligence is not “knowing more.” It is attending to the right subset. intelligence as selective relevance 6. AI as search Generation can be understood as search through possible continuations, possible plans, possible answers, possible programs, possible explanations. The model proposes candidates. Decoding chooses among them. External systems can expand, rank, prune, retry, or verify. This abstraction explains why “one-shot answer” is weak. A stronger system treats output as candidate generation inside a search process. AI as proposal engine inside a search space 7. AI as amortized reasoning A model often appears to reason instantly because it has compressed many previous reasoning patterns into its weights. Instead of solving every problem from scratch, it uses learned shortcuts. That is amortization: expensive training produces cheap inference. This is why models can be fast and impressive, but brittle on problems outside their learned distribution. training stores reusable cognitive shortcuts 8. AI as translation between representations AI is often not “answering.” It is translating. Natural language to code. Image to caption. Question to SQL. Messy notes to outline. Goal to plan. Plan to action. Error log to diagnosis. Data to explanation. Text to UI. UI to state. This is broader than the IR/runtime idea. The key is not execution. The key is representation change. intelligence as representation transformation 9. AI as constraint satisfaction A model is constantly satisfying constraints: The grammar must work. The answer must fit the prompt. The style must match. The facts should cohere. The schema must validate. The task goal should be met. The next token must be likely. More constraints usually produce more useful output, as long as they are not contradictory. This reframes prompting as constraint design. prompting as constraint programming in natural language 10. AI as an uncertainty engine A model does not truly “know” one answer internally. It has distributions over possible answers. Confidence, ambiguity, hallucination, creativity, and disagreement all come from this. Bad systems collapse uncertainty too early. Good systems preserve uncertainty until verification or choice. Examples: “Maybe these three causes.” “Here are candidate interpretations.” “This field is unknown.” “This action needs confirmation.” “These outputs have ranked likelihood.” intelligence as managing uncertainty, not eliminating it 11. AI as a critic A model does not only generate. It can evaluate. It can score whether an answer follows instructions, whether a plan is plausible, whether a UI component matches intent, whether a command looks risky, whether a summary preserves meaning. Generation and criticism are different roles. The same architecture can be used as a proposer or as a judge, but those are different abstractions. AI as generator versus AI as discriminator 12. AI as feedback control Once AI interacts with the world, the key abstraction becomes control. There is a goal, an action, an observation, and an error signal. The system acts again to reduce the gap between current state and desired state. This is different from text generation. It is closed-loop behavior. A non-agentic model produces an output. A control system keeps adjusting until the state changes. agency as closed-loop error correction 13. AI as policy A policy maps states to actions. This abstraction comes from reinforcement learning, but it applies broadly. Given the current situation, what should be done next? A language model can be seen as a policy over tokens. An agent model can be seen as a policy over actions. A router can be seen as a policy over modules. A UI model can be seen as a policy over components. intelligence as choosing the next move 14. AI as affordance detector An affordance is a possible action implied by the environment. A button affords clicking. A text box affords typing. A file affords opening. A sentence affords summarizing. A table affords querying. A contradiction affords correction. Models are useful when they detect what can be done with a given object or situation. understanding as seeing possible operations 15. AI as ontology builder The model does not just process facts. It carves the world into categories. It decides what the objects are, what relations matter, what level of description is appropriate, and what distinctions are meaningful. This is huge. Before solving anything, the system frames the domain: Is this a planning problem? A debugging problem? A design problem? A routing problem? A classification problem? A conflict between constraints? intelligence as choosing the right ontology 16. AI as compression of procedures into skills Repeated behavior can be compressed into a reusable unit. At first, the system reasons step by step. Later, the same pattern becomes a skill, macro, function, habit, or policy. This is how intelligence becomes efficient. Deliberate reasoning hardens into automatic procedure. learning as turning deliberation into reusable action 17. AI as boundary negotiation AI systems constantly negotiate boundaries: What is inside the model versus outside? What is language versus action? What is user intent versus system policy? What is memory versus context? What is suggestion versus execution? What is uncertain versus confirmed? What is generated versus retrieved? Many failures come from bad boundary placement. system design as deciding where boundaries live 18. AI as scale-shifting AI can move between levels of abstraction. It can summarize a thousand details into one concept. It can expand one concept into detailed steps. It can turn a principle into examples. It can turn examples into a rule. It can turn a rule into a procedure. This is one of the most important general capabilities. intelligence as moving between compression levels 19. AI as style-transfer over meaning Style is not superficial. Style changes which features are emphasized. A legal style foregrounds definitions and risk. A technical style foregrounds mechanism. A poetic style foregrounds association. A terse style foregrounds selection. A tutorial style foregrounds sequence. The same content passed through different styles becomes different cognitive machinery. style as an information filter 20. AI as meaning-preserving distortion Every model output distorts the input. The question is whether the distortion preserves what matters. Summarization distorts by compression. Translation distorts across languages. Explanation distorts by simplification. Classification distorts by discretization. Visualization distorts by projection. Analogy distorts by mapping. This is a clean abstraction: AI transforms information by controlled distortion 21. AI as synthetic sensor A model can act like a sensor for things that are not directly measured. It can detect tone, intent, topic, risk, ambiguity, contradiction, relevance, salience, sentiment, genre, structure, or implied goal. These are not physical signals. They are semantic signals inferred from data. AI as a sensor for latent properties 22. AI as semantic glue AI is useful because it can connect systems that were not designed to talk to each other. Humans speak loosely. Machines require formality. Databases have schemas. UIs have components. Files have formats. APIs have contracts. AI sits between incompatible representations and makes them interoperable. AI as adapter between mismatched worlds 23. AI as entropy manager Raw possibility is high entropy. A model reduces it. A blank page has infinite possibilities. A prompt narrows them. A schema narrows them further. A plan narrows them into sequence. A final answer collapses them into one path. Creativity increases useful entropy. Execution reduces entropy. AI systems alternate between expanding and collapsing possibility 24. AI as a mirror with transformation A model often reflects the structure of the input back at you, but transformed. Ask vaguely, get vague shape back. Ask with structure, get structure back. Ask with assumptions, get those assumptions amplified unless challenged. Provide examples, get pattern continuation. This is not just “garbage in, garbage out.” It is more like: the model amplifies the latent structure of the input 25. AI as a coordination layer AI can coordinate between agents, tools, documents, users, tasks, goals, and representations. It does not necessarily do the work itself. It decides who or what should do which part, in what order, under which constraints. This is orchestration at the semantic level.
